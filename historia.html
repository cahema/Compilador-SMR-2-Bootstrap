Los primeros compiladores


En los orígenes de la informática no existian lenguajes de programación de alto nivel como los que conocemos hoy en día, en su lugar los programas se escribían a mano en tarjetas perforadas o en código máquina ya que para que existan lenguajes de alto nivel tiene que existir la forma de convertirlos a código que el ordenador pueda entender. Por ello hacía falta un programa capaz de convertir este código, esto es lo que se llama un compilador. El primer concepto de compilador lo tuvo Corrado Böhm, un profesor, matemático e informático teórico que escribió en su tesis, Calculatrices digitales du déchiffrage de formules logicomathématiques par la machine même dans la conception du programme el concepto de un programa capaz de convertir código de alto nivel a código máquina y a la vez el usar ese programa para compilar el propio compilador en un proceso llamado bootstrapping, en el cuál el compilador se compila a si mismo.

Bootstrapping


El bootstrapping fue uno de los grandes avances de los compiladores, gracias a este proceso cuando se creaba una nueva arquitectura de hardware había que al principio crear un compilador mínimo usando código máquina y este a su vez se usaba para compilar nuevas versiones del compilador, esto hacía que hubiese mejoras incrementales, las primeras versiones de estos compiladores son simples y sirven únicamente para compilar código pero las versiones futuras arreglan errores y optimizan el código permitiendo mejoras incrementales.

Compiladores modernos

GCC

GNU Compiler Collection es un conjunto de compiladores creados por el proyecto GNU

Clang

Clang es un front end de compilador para los lenguajes de programación C, C++, Objective-C y Objective-C++

Cython

Cython es un compilador estático optimizado tanto para el lenguaje de programación Python como para el lenguaje de programación extendido Cython (basado en Pyrex)
